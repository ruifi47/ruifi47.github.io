## __Introduction__

Recently, I participated in the [H7CTF International](https://ctftime.org/event/2491) with [ransoware](https://ctftime.org/team/218596), placing 14th out of 527 teams. Here are the writeups for some of the challenges that I solved during the CTF.

## __Web__

### __No Paste__

__Description__
> "If you spend too much time thinking about a thing, you'll never get it done. Paste it up, cut it out, and just do it" â€” Unknown

In this challenge, we have a webpage with an input box that is disabled (it stops the user from writing anything in it) and a submit button.
If we click the submit button we get a message saying `Invalid input!`, so we need to find a way to bypass this restriction and send a valid input.
Using the browser devtools, it is possible to see the requests made by the application.

![No_Paste-2](No_Paste-2.png)
_Challenge webpage and devtools network activity_

### __Code Breaker__

__Description__
> Leaked creds in hand: { 'user1': 'password1', 'user2': 'password2', 'user3': 'password3' } But the Admin panel remains out of reach. "A higher access level requires a sharper mind."

After accessing the challenge URL we are presented with the following login page.

![Code_Breaker](Code_Breaker.png)
_login page_

From the challenge description, we can predict that we need to find a way to reach the admin panel. The challenge description also gave us credentials to access a few user accounts, so we can go through all them to see if we can collect some useful information. After exploring the 3 users accounts, none of them contained useful information after the authentication in their panels (they were all identical to the image below). Since there is not any interesting functionality inside the authenticated panels to test, we started to look into how the sessions were established. Let's check cookies.

![Code_Breaker-2](Code_Breaker-2.png)
_user1 authenticated page_

The cookie looks like a base64 encoded string. Taking the user1 cookie and trying to decode it, we confirm that it is indeed base64 encoded.

```shell
$ echo dXNlcjEyNjg= | base64 -d
user1268
```

By decoding other cookies we noticed that the cookie values follows a pattern. It is constructed with the username concatenated with 3 random numbers, and then base64 encoded. From this info we could probably attempt to get the admin cookie by assuming the username is admin and bruteforce the last 3 characters. However, after setting the cookie as a non base64 encoded value, we crashed the application.

![Code_Breaker-4](Code_Breaker-4.png)
_crashing the application_

The crash revealed some sensitive server side code that we could leverage to login as admin. As we can see in the image above, the application checks if the decoded cookie value equals `admin9876543` and, if this condition is true, we should have privileged access to the admin panel. So let's base64 encode the value `admin9876543`, set it as our cookie and hopefully we get access the `/admin` endpoint and find the flag.

![Code_Breaker-6](Code_Breaker-6.png)
_`/admin` endpoint with the flag_

__Flag__: `H7CTF{cr4ck_d_c00k13_b64dEc0de}`

### __Paper Chase__

__Description__
> I fixed the leaks! Think you can outsmart the new defenses? Give it another shot!

The web application for this challenge is a newspaper viewer where the newspapers html are embedded in an iframe. We also have 2 buttons to see the previous or next newspaper.

![Paper_Chase](Paper_Chase.png)
_web application main page_

Since there are not another interesting functionalities to test, we looked into the webpage source and discovered that we can control the `file` parameter value of the endpoint `view.php?file=`.

![Paper_Chase-1](Paper_Chase-1.png)
_web application source_

Since the application allow us control the file parameter, it can potentially be vulnerable to [path traversal](https://portswigger.net/web-security/file-path-traversal). So we tried some classic `../` payloads variations in an attempt to leak some sensitive files stored outside the web root folder and reaching the one that contains the flag. However, we quickly noticed that the application was filtering `..`, substituting it for `#`. One way to bypass the filtering is using the `file://` URI scheme, which allowed to successfully view the contents of `/etc/passwd` file with `/view.php?file=file:///etc/passwd` and get the flag to solve the challenge.

![Paper_Chase-3](Paper_Chase-3.png)
_Leaking the flag_

__Flag__: `H7CTF{L0c4l_F1l3_1nclusi0n_4t_R3qU1r3}`

### __Bitwise Mystery__

__Description__
> This challenge obscures the flag using a combination of bit shifts and character manipulation. Discover how the paired characters are transformed and recover the hidden message!

To solve this challenge we have 2 files. A compiled bytecode file `chall3.pyc` and a text file `out3.txt`.

First, we have to get the python3 source code from `chall3.pyc`. For this task, we used the online [PyLingual Python Decompiler](https://pylingual.io/), which gave us the following python code:

```python
flag = 'REDACTED'

    n += '*'

    a, b = (ord(n[i]), ord(n[i + 1]))
    m.append((a << c) + b)
print(m)
```

### __Scrambled Pathways__

__Description__
> In this challenge, characters follow a hidden pattern as they traverse columns. Figure out how to untangle the message and reveal the flag!

This challenge provided 2 files to solve it, a binary file `chall1` and a text file `out1.txt`.

First, we opened the binary in [Ghidra](https://ghidra-sre.org/) to have a look at the decompiled code. It was a C program that basically performs some transformations to the original flag and outputs the shuffled flag.

The text file contained the output value of the shuffled flag characters.

From the output above, we recovered the original input (Flag) with the following python script:

```python
def reverse_transformation(output):
    part_len = (len(output) + 2) // 3

    part1 = output[:part_len]
    part2 = output[part_len:2 * part_len]
    part3 = output[2 * part_len:]

    input = []

    for i in range(part_len):
        if i < len(part1):
            input.append(part1[i])
        if i < len(part2):
            input.append(part2[i])
        if i < len(part3):
            input.append(part3[i])

    return ''.join(input)

out = "HT859092947c87F4cff7727181C{872a2ba640}"
print(reverse_transformation(out))
```
